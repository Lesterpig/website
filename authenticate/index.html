Authenticate Mopidy with music services.

Some services require you to get authentication tokens for Mopidy so that it can connect to your service accounts. This page helps you on the way.

Photo by Sergiu Bacioiu

SoundCloud

If you want to use the SoundCloud extension, you must first authenticate your Mopidy installation to access your SoundCloud account.

  1. No information is shared with the Mopidy web site.

  2. Add the token to the soundcloud/auth_token config value in your mopidy.conf file:

    [soundcloud]
auth_token = ...token goes here...

For more information see the Mopidy-SoundCloud docs.

Spotify

If you want to use the Mopidy-Spotify and/or Mopidy-Spotify-Web extensions, you must first authenticate your Mopidy installation to access your Spotify account. Once authenticated, your credentials will appear below.

  1. Your Spotify login details are never shared with this site.

  2. Add the credentials to the spotify config section in your mopidy.conf file:

    [spotify]
client_id = ...client_id will appear here...
client_secret = ...client_secret will appear here...

  3. If also using Mopidy-Spotify-Web then also add the same credentials to the spotify_web config section in your mopidy.conf file:

    [spotify_web]
client_id = ...client_id will also appear here...
client_secret = ...client_secret will also appear here...

For more information see the Mopidy-Spotify docs and Mopidy-Spotify-Web docs.
